User Permissions page

Assign RC-WebView access permissions to users using the User Permissions page.

Access the User Permissions page in one of two ways: 

  • Add a new user through the Add User wizard and set the user permissions in the wizard.
  • Click a user's name in the System Tree, then click the Permissions button.

If a user has been assigned a user role, the user may be inheriting permissions. Inherited permissions are labeled with Inherited Role . Hover the cursor over the icon to see which role is providing this setting.

Use the following check boxes to assign permissions for the current user:

  • Administer — Allows the user to access and configure the main features within their Enterprise Website.
  • Modify — Allows the user to view and/or modify the value or state of the objects accessed through the Navigation Tree of an Enterprise Website.
  • Alarms — Allows the user to view and acknowledge alarms from all systems connected to the Enterprise Website.
  • Access — Allows the administrator to restrict access of the user based on level and role.
  • Enterprise Schedules — Allows the user to create, modify, and configure after hours Events and Enterprise Schedules.

Alarms

Access

Audit Trail

View the Audit Trail.

End-user Values

Modify object values in System Groups, which are configured as End-user in RC-Studio. If a point is set to End-User in RC-Studio, an RC-WebView user must have this access enabled to edit the object.

Operator Values

Modify Operator values in System Groups which are configured in RC-Studio. If an object is set to Operator in RC-Studio, an RC-WebView user must have this enabled to edit the object.

Access Level

The access level must be equal to or higher than the RC-Studio security level for the Operator to edit the object value.

Auto/Manual

Enables the user to place the object into auto, manual, or timed override.

Must Enter Change Reason

When selected, requires the user to enter a reason when modifying an object value on a System Group. These change histories appear when viewing the Audit Trail worksheet.

Auto Log Off

Sets the length of time of inactivity (default: 15 minutes) after which the user is automatically logged off of the Enterprise Website.

Set System Date and Time

Change the date and time of the system within the Enterprise Schedule. This applies to all systems within the user's Enterprise Website.

Programs

Access Control-BASIC programs.

Custom Help Files

Custom help files can only be added in the user Permissions page while creating a new user. Use this option to Upload any special files that may be helpful to the user, for example, a floor plan or sequence of operation. This will replace the Reliable Controls help files for this user. Only one file can be uploaded at a time. Supported file types include; pdf, html, doc, docx, odt, and txt. The Gallery button appears if there have been previously uploaded help files. Select Gallery to see a list of previously uploaded custom help files to select. To clear the custom help files and provide the user with the Reliable Controls help files again, clear the Custom Help File text box and click .

Note: This will replace the existing help files in RC-WebView for this user.

Enterprise Schedules

Create/Modify After Hours Events

Approve After Hours Events

Allows the user to approve or reject After Hours requests .

Pop-up After Hours Notifications

Provides pop-up notifications for After Hours Events. If the permission is disabled, these notifications are not shown to the user.

Create/Modify Default Schedule

Allows the user to create or modify the default Schedule in Enterprise Schedules.

View After Hours Report Summary

Provides the user with access to the After Hours Report Summary.

Configure Event Priority

Allows the user to configure Event priorities in Enterprise Schedules.

Configure After Hours Event Reporting

When selected, allows a user to opt out of Special Events being recorded in the After Hours Report.
