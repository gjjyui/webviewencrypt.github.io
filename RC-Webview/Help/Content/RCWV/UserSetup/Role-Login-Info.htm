Role Login Info page

Use the Role Login Info page to specify a user role's logon information. If a role is applied to a user, this logon information will be inherited by that user.

To display the Role Login Info page, click Users in the System Tree, click Role List on the Login Info page, and then click on a role name.

Select or clear the check box next to each option to enable or disable this feature in the role.

The following describes the options on the Role Login Info page:

Enabled/Disabled

Enable or disable a role. A disabled role can still be applied to users, but the settings won't take effect until the role is enabled.

Navigation Tree

Select the check box to provide the user role control over which navigation tree that the user can access. If the check box is not selected, it has no effect.

After Hours Template

Select the check box to provide the user role control over which After Hours Report Template the user can access. If the check box is not selected, it has no effect.

Home Group

Home Group settings determine what displays when a user first logs on.

  • Log in to  — Select Navigation Group or System Group to display their respective options.
  • Navigation Group  — Displays when you select Navigation Group in the Log in to option. Select a Navigation Group in the list to use as the initial page.
  • System and System Group — Displays when you select System Group in the Log in to option. Select an available system from the System list and then select a System Group for that system from the System Group list.

myControl Home Group

myControl Home Group settings determine the System Group that displays when a user logs in using the myControl app.

  • System — Select a system from the drop-down list.
  • System Group — Select a System Group available on the selected system.

Permitted Systems

Determines the systems the user can work with to view and manage data. Click to choose between All Systems or one or more individual systems that have been defined in RC-WebView.

Force All Systems

Enforces that users inheriting the role will have access to all systems regardless of what other inherited roles are granting the user.

Permitted Schedules

Determines the Schedules that the user can access and manage. The user will only have access to the selected Schedules within that Enterprise Schedule.

Force All Schedules

Enforces that users inheriting the role will have access to all Schedules within Enterprise Schedules regardless of what other inherited roles are granting the user.
