Title 21 CFR Part 11 compliance

The Code of Federal Regulations Title 21 Part 11 Electronic Records; Electronic Signatures (CFR Title 21 Part 11) defines the criteria under which "records in electronic form that are created, modified, maintained, archived, retrieved, transmitted, or signed are considered to be trustworthy, reliable, and generally equivalent to paper records and handwritten signatures executed on paper." (Source: U.S. Food & Drug Administration, CFR - Code of Federal Regulations Title 21.)

RC‑WebView supports Title 21 CFR Part 11 with options on the Enterprise Website Security page that pertain to:

  • Enforcing approval for changes.
  • Enforcing password expiry and configuration options for user lockout.
  • Applying a watermark to printed system information.
  • Automatic availability of a site signature certificate for digital signing purposes.

These options are described in further detail in Security settings for Title 21 CFR Part 11 compliance.

The following features are implemented by default in support of Title 21 CFR Part 11:

  • Validation of audit log entries.
  • Context filtering of the audit trail.
  • Context filtering of alarms and alarm history.
  • Digital signing of Excel file exports.

These features are described in more detail in Features supporting Title 21 CFR Part 11 compliance.

Important: After an RC‑WebView installation is commissioned to support Title 21 CFR Part 11, no other workstation software (RC‑Studio, RC‑Archive Database Utility, and RC‑Toolkit) is permitted on site, and the myControl application cannot be used. Otherwise, compliance is negated.

Security settings for Title 21 CFR Part 11 compliance

The Enterprise Website Security page includes options that provide compliance with Title 21 CFR Part 11.

Enforce change approval

The Change Approval area enforces the use of an approval password with a reason for the change to the connected system.

Figure: Change approval options

Important: Before you enable this option, ensure there are a minimum of two administrators with Administer Systems and Administer Users permissions. When either administrator updates the change approval, grace period or lockout settings on the Website Security page, their approval, a reason for the change, and the secondary approval of another administrator is required. In addition, if an administrator changes log on or approval passwords for themselves or other users, another administrator needs to provide secondary approval.

Figure: Administrator approval and reason for the change

Figure: Secondary administrator user name and approval password

The Enforce Change Approval option changes the Login Info page for users. Approval and Verify Approval Password input boxes appear on the page along with the option to require the user to reset their approval password when they log on.

Figure: Approval password area on a user Login Info page

When a user saves a change to the system, they are prompted to enter their approval password and a reason for the change.

Also enabled with the Enforce Change Password option is the grace period setting: Approval password required for system changes every x minutes. When a user saves and approves a change, they can make further changes within the grace period without reentering their approval password. When the grace period expires, they need to use their approval password again. For instance, if the setting is 1 minute, and an administrator or user enters their approval password and reason for a system change, they can make additional system changes within that minute without reentering their approval password and a reason.

However, for Title 21 CFR Part 11 compliance, the setting needs to be 0 minutes to enforce an approval password and reason for every change.

Enforce password expiry and lockout options

The Enterprise Website Security page includes areas where you can enable the password expiry function and set the options for locking a user out of the application.

Figure: Password Expiry and Lockout Settings options

Password expiry and lockout options mitigate the risk of compromised passwords by narrowing the time window for accessing the system and blocking unauthorized attempts to gain access to an authorized user session.

The Enforce Password Expiry option allows you to specify the number of days until a log on password expires and a new password is required.

Note: The Password Expiry options are not applicable to Active Directory users or groups because the passwords are not controlled through RC-WebView.

The Lockout Settings options allow you to specify the number of log on attempts before a user is locked out of RC‑WebView and to set the duration of the lockout.

These options gives administrators the ability to customize various levels of security as dictated by site requirements.

Watermark applied to printed system information

When you select the watermark option, an image is applied as a watermark to all files printed from the system using Options > Print.

Figure: Watermark option and the default image name

You can specify an image to replace the default Reliable Controls watermark image. The image is automatically resized and repeated across the page at an angle. The image transparency is adjusted to ensure that the printed content remains visible.

The option to apply a watermark is available at all times and is not dependent on the change approval settings.

Automatic creation of a Site Signature Certificate

The signature certificate is used to sign and validate audit logs, and to digitally sign Excel exports. RC‑WebView automatically creates a default signature certificate that is identified within the Signature Certificate area.

Figure: Signature Certificate area showing the default certificate name

To select a different certificate, click Certificate and choose an available certificate in the Select Signature Certificate dialog box. The dialog box lists the certificates in the Local Computer\Personal certificate store on the server that hosts RC‑WebView. Only certificates with a private key are shown in the dialog box.

Figure: Select Signature Certificate dialog box

Features supporting Title 21 CFR Part 11 compliance

Additional support for Title 21 CFR Part 11 compliance is provided by default and encompasses validation of audit log entries, context filtering with restrictions on viewable data and on the exporting of signed Excel files when the enforce change approval option is in effect. These items are described in more detail in the following sections.

Validation of audit log entries

Log entries in the audit trail are digitally signed to indicate they are not manipulated by an outside source.

When you view the entries in the log, a check mark in the Validated column indicates the entry is validated and signed using the signature certificate credentials. For example, when a user changes the state of an object in a System Group, they are prompted for their approval password. The log entry in the audit trail includes their username in the Name column and a check mark for the entry in the Validated column. When an administrator changes a setting that requires another administrator's approval, the audit trail includes the username of the administrator who made the change in the User column, the username of the approving administrator (the secondary approver) in the Approval User column, and a check mark in the Validated column.

Context filtering of the Audit Trail

Audit log entries in the audit trail are automatically filtered in relation to the object currently displayed. For example, context filtering occurs when you open the audit trail while viewing a System Group. In this case, only the audit log entries for the objects referenced in the System Group are shown. To show all entries in the audit trail, click Clear All Filters.

When the change approval option is enabled, the Clear All Filters function is not available. To filter the entries displayed within the context, a user can click Filter in the toolbar. To clear the filtered context entries, the user can click Filter again, click Clear in the Filter Records dialog box, then close the dialog box. Audit trail filtering is not changed for users with Administer System and Administer Users permissions.

Context filtering of alarms and alarm history

The Alarms and the Alarm History pages are automatically filtered to show only the entries for the device being viewed. For example, when you view a system group named 1000GRP1, only the alarm entries for device 1000 are shown. The title identifies the worksheet name and the device name. For example. when you view the alarm history for System Group 1000GRP1 for a MACH-ProWebCom with a device name set to ControllerA, the title is Alarm History - ControllerA (1000).

When the change approval option is enabled, the Clear All Filters function is not available to display all entries in the Alarms and the Alarm History pages, except for users with Pop-Up Alarms permission. To filter the entries displayed within the context, a user can click Filter in the toolbar. To clear the filtered context entries, the user can click Filter again, click Clear in the Filter Records dialog box, then close the dialog box.

Exporting Excel files

When you select the Enforce Change Approval option, the audit trail can be exported only as an Excel file. The option to export as a CSV file is not available. The Excel file contains a Data sheet and a Signature sheet.

The Data sheet includes user readable data corresponding to the entries in the Audit Trail.

The Signature sheet includes the following columns:

  • Validated — Validated entries contain True in the column. Entries with False in the column have not been validated.
  • Raw Serialized Log — Non-locale-dependent audit log data used to create the signature hash.
  • Signing Algorithm — The name of the algorithm used to sign the serialized log into the signature.
  • Digital Signature — The hash that results when the serialized log data is signed with the signing algorithm using the site's signature certificate.
  • Public Key — Used by an auditor to verify the signature was created with the corresponding private key.
  • Signature Certificate Thumbprint — The unique identifier for the certificate.

Using the information contained in the signature certificate, an auditor can validate the authenticity of the entries.

When you open the Excel file you downloaded, it may open in Protected view. Go to File > Info and click Enable Editing for the Protected View item.

The following notations appear at the top of the spreadsheet:

  • MARKED AS FINAL An author has marked this workbook as final to discourage editing — This indicates that the Excel file is protected.
  • SIGNATURES This document contains valid signatures — Indicates that the site's signature certificate has been used to sign the entries.

These notations are also contained on the Excel Info tab (File > Info).

If these notations do not appear at the top of the spreadsheet, go to File > Info and click Enable Editing for the Protected View item.

Figure: The Info tab for a read-only (protected) Excel file

If the user's computer does not already trust the certificate used to sign the Excel file, the user can add the signature and trust the certificate with the following steps:

  1. When you open the read-only (protected) document, click View Signatures.
  2. In the Signatures area on the right, click the certificate name to activate the drop-down menu and click Signature Details.

  3. Click Click here to trust this user's identity.

    The Signature Details dialog box indicates the signature is valid, the content has not changed, and the signer's certificate is valid.
