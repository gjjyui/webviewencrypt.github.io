Enterprise Website Properties

You can modify the following options when you click Properties on the Enterprise Website Settings page:

Default Options

The following settings configure the color theme, name, and URL of the Enterprise Website.

Default Theme

The color scheme to be applied to the website.

Name

The name to be displayed in the banner of the Enterprise Website (if configured) and in the System Tree.

Base URL

The base of the URL for the end user's URL path.

URL Path

The optional end portion of the end user's URL path.

Site URL

The combined total of the URL path, BASE URL/ URL Path

Default Site

Select the check box to make the Site URL the default site for RC-WebView. If no valid Enterprise Website is specified in the URL, the user will be redirected to this Enterprise Website.

SMTP

The following settings configure SMTP for email notifications.

Enable SMTP

Enable or disable SMTP.

Server

The email server from the drop-down list or select other to enter different email server information. If you select Gmail, Yahoo, or Hotmail/Live/Outlook.com, the next three sections are automatically completed.

Server Address

The email server address. This information is provided by your email provider.

Port Number

The Port Number for the email server. This information is provided by your email provider.

Security Type

The Security Type for the email server. This information is provided by your email provider.

Sender Email

The email address of the user. Enter a valid email address for validation.

Username

Email user name (Optional).

Password

Email password (Optional).

Test

Used to test the configuration of the connection to the email server. A green notification stating Test Successful appears if the connection settings are correctly configured.

Manage Certificates

Opens a dialog box used to specify a digital security certificate. This is unnecessary if using Gmail, Yahoo, or Hotmail/Live/Outlook.com.

Active Directory

The following settings configure automated user credential management for access to the Enterprise Website using Microsoft Active Directory. When you complete these settings, you need to add the users with their Active Directory credentials when you complete their login information for RC‑WebView.

Enable Active Directory Authentication

Authenticates user access using the domain's Lightweight Directory Access Protocol (LDAP) server. For more information, see the Microsoft Developer Network (MSDN). When you select this option, you can specify up to three additional LDAP servers for redundancy in the event an LDAP server stops running. Click Add beside the Active Directory Provider option.

Enable Active Directory Connection through SSL

Encrypts all web traffic to and from the LDAP server using a Secure Sockets Layer (SSL).

Active Directory Provider

The IP address or host name of the domain's Active Directory server. The default Active Directory port number is 636. If you are using this port number for another application, you need to append the port number that you want to use for your Active Directory server to the IP address or host name with a colon and the port-number, that is, :port-number.

When you click the Add icon, another Active Directory Provider input box is displayed. You can add up to three additional LDAP servers.

Domain Name

The domain name that will be used for Active Directory authentication.

Domain User

A user name within the Active Directory domain that has at minimum the "Read Member Of" permission to user objects in the domain. Any "system admin" account may be used. This permission must be set by the Active Directory System Administrator for your domain.

Domain Password

The password of the user that is named in the Domain User text box.

Alarm Emails

The following settings configure the content included in the subject and body of alarm notification emails. Click Configure Distribution to configure the alarm recipients.

Alarm Emails — Subject area

In the Subject area, select one or more check boxes to include the respective item in the email title. The following items are available for selection:

Site Name

Displays the name of the Enterprise Website causing the alarm.

System Name

Displays the name of the system causing the alarm.

Object Name

Displays the name of the object causing the alarm.

Message

Displays any message that may be included with the alarm.

Alarm State

Displays the current state of the alarm.

Alarm Type

Displays the type associated with the alarm (System, General, or one of 62 user-defined types).

Alarm Emails — Body area

In the Body area, select one or more check box to include the respective item in the email body. The following items are available for selection:

Site Name

Displays the Enterprise Website causing the alarm.

System Name

Displays the connected system that generated the alarm.

Source

Displays the name and device instance (or RCP controller number) of the BACnet device (or RCP controller) that generated the alarm.

Object Name

Displays the name of the object causing the alarm.

Message

Displays the message associated with the alarm. Messages can be up to 69 characters.

Alarm State

Displays the current state of the alarm.

Alarm Values

Displays the values of the alarm.

Status Flags

Displays the status flags of the alarm.

Alarm Time

Displays the date and time that the alarm was generated at the controller level.

Alarm Type

Displays the type associated with the alarm (System, General, or one of 62 user-defined types).

Alarm Priority

Displays the priority of the alarm, as defined in the Control-BASIC program, alarm configuration, or intrinsic alarm setup. BACnet Notification Classes map 1-to-1 with Reliable Controls alarm priorities.

Action

Displays which alarm action was triggered using the Control-BASIC alarm statement: #G or #R, and is left blank for intrinsic alarms unless they include a message with a #G or #R action. If you select Action to be included in the email, a Log On to Webview icon links to the action for the alarm in RC-Webview or to the Enterprise Website.

Log on Link

This option is selected by default and indicates that a link to RC‑WebView Log On page is included in the body of an alarm notification email. Clear the checkbox to exclude the link from the email notification.

Banner Settings

The following settings configure the appearance of an optional logo and/or banner displayed at the top of the Enterprise Website.

Show Website Name in Banner

Indicates whether or not the website name appears in the banner at the top of RC-WebView.

Current Logo Image - Browse

Browse lets you select a logo image from your files to be displayed in the browser header. The selected image is re-sized vertically to a height of 49 pixels keeping the original aspect ratio and it is horizontally left-aligned within the banner area of the browser header.

Current Logo Image - Clear

Clear removes the uploaded image and restores the default logo image.

Current Logo Image -Gallery

Gallery lets you view a gallery of previously uploaded logo images.

Current Banner Image - Browse

Browse lets you select a banner image from your files to be displayed in the browser header. The selected image is re-sized vertically to a height of 60 pixels keeping the original aspect ratio and it is horizontally left-aligned within the banner area of the browser header.

Current Banner Image - Clear

Clear removes the uploaded image and restores the default banner image.

Current Banner Image - Gallery

Gallery lets you view a gallery of previously uploaded banner images.
