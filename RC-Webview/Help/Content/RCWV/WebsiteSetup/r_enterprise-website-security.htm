Enterprise Website Security

You can modify the following options when you click Security on the Enterprise Website Settings page. RC‑WebView supports Title 21 CFR Part 11 with these options. See Title 21 CFR Part 11 compliance for more information.

Change Approval

Enforce Change Approval

Enforces the use of an approval password with a reason for the change to the connected system.

Important: Before you enable this option, ensure there are a minimum of two administrators with Administer Systems and Administer Users permissions. When either administrator updates the change approval, grace period or lockout settings on the Website Security page, their approval, a reason for the change, and the secondary approval of another administrator is required. In addition, if an administrator changes log on or approval passwords for themselves or other users, another administrator needs to provide secondary approval.

When this option is enabled, Approval and Verify Approval Password input boxes appear on the Login Info page for each user along with the option to require a new approval password when the user logs on to RC‑WebView. In addition, the CSV export option is removed from all pages where it was available. The XLS Export option remains on those pages.

Approval password required for system changes every x minutes

Indicates a grace period in minutes before the user is prompted to enter their approval password after their last approval. When the grace period ends, they need to use their approval password again. If the option is set to 0 minutes to comply with Title 21 CFR Part 11, the user is prompted for their approval password for each change they make in a connected system.

Password Expiry

Note: The Password Expiry options are not applicable to Active Directory users or groups because the passwords are not controlled through RC-WebView.

Enforce Password Expiry

Select the check box to indicate that passwords for users accessing the website expire after a specified number of days.

Password expires every

Enter the number of days before the passwords expire.

Lockout Settings

Number of attempts before lockout

Enter a value for the number of permitted logon attempts. The value is set initially to 5 when you select a lockout option.

No Lockout

Indicates that users are not locked out of the website regardless of the number of logon attempts.

15 Minute Lockout

This is the default lockout option and it indicates that users who exceed the value specified for Number of attempts before lockout, are locked out of the website for 15 minutes. Users can attempt to log on again after 15 minutes but they will be locked out again if they exceed their allowed number of attempts.

Requires Administrator to Unlock

Indicates that users who exceed the value specified for Number of attempts before lockout are locked out of the website until the website Administrator clears the Account is locked out check box setting on the user's Login Info page.

Watermark

Select the check box to apply a watermark to all files printed from the system using Options > Print. The image is resized and applied to the each page at an angle. The image transparency is adjusted to ensure the information in the file to be printed remains visible.

... (browse)

Select an image from your files to use as the watermark in place of the Reliable Controls watermark image. Your selection is automatically added to the image gallery.

Default

Resets the watermark image to the default image provided by RC‑WebView.

Gallery

Opens the Gallery dialog box, which lists previous uploaded images. Select an image and click OK to use it as your watermark or click Delete to remove the image from the gallery.

Signature Certificate

The signature certificate is used to sign and validate audit logs, and to digitally sign Excel exports. RC‑WebView automatically creates and identifies a default signature certificate within the Signature Certificate area. To select a different certificate, click Certificate and choose an available certificate in the Select Signature Certificate dialog box. The dialog box lists certificates found in the Local Computer\Personal certificate store on the server that hosts RC‑WebView. Only certificates with a private key are shown in the dialog box.
