PRINT

Initiates the printing of the specified string. This is not supported by the MACH-ProAir, MACH-ProZone, or any controllers running version 8 firmware.

Statement

Initiates the printing of the specified string. Must be used in conjunction with the PRINT-AT statement to specify the controller with an attached printer.

Format

PRINT “string” [;] [BEEP] [TIME] [USER-A] [USER-B] [$XX] [point] [;]

Where

string is the message to print. Each message must be enclosed in “ “..

; is a formatting element used to suppress the sending of a carriage return/line feed pair at the end of a line.

This character is also used to separate multiple print arguments within one PRINT statement.

 

The following terms are used in conjunction with the PRINT statement to provide information or initiate actions.

When used in a PRINT statement, do not enclose them in the double quotes for the remainder of the message;

use a semi-colon to create a continuation of the same line between multiple arguments.

BEEP

Causes the printer to beep.

TIME

Prints current date and time.

USER-A

Prints the name of the user logged on at port A (MACH-Global controller only).

USER-B

Prints the name of the user logged on at port B.

XX

Where XX is a HEX number that represents the character code to be printed.

System Points or Local Variables

When these elements are included in a PRINT message, the current value of the system point or local variable is printed.

Example 1

10 PRINT-AT All

20 IF+ FIRE THEN GOSUB 100

30 END

100 REM Fire instructions follow

110 PRINT “The following procedure should
    now be done.”

120 PRINT “1. Call Fire Department.”

130 RETURN

Example 2

10 PRINT-AT 2

20 IF+ USER-A THEN PRINT USER-A ; “has signed-on at: ” ; TIME

30 IF+ ROOM TEMP > MAXIMUM THEN GOSUB 100

40 IF INTERVAL( 01:00 ) THEN PRINT “Building low is” ; L ; “Time” ; TIME

50 END

100 PRINT “Value of Room Temperature is:” ; ROOM TEMP ; “at” ; TIME ; BEEP ; BEEP ; BEEP

110 PRINT “Message 1”

120 PRINT “Message 2”

130 RETURN

The PRINT statement must be used in conjunction with the PRINT-AT statement to specify the controller required to complete the print operation, or the controller with the printer attached. If the PRINT-AT statement is not used, the messages are not printed.

Use the IF+ and IF- statements with a PRINT statement to avoid printing the same string repeatedly.
