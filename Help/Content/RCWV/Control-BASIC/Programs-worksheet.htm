Programs worksheet

Use the Programs worksheet to add and modify Control-BASIC programs on a controller.

Access the Programs worksheet by clicking on a controller in the System Tree, then click Programs in the flyout menu.

Note: You can add a link to the Programs worksheet in a System Group in RC-Studio.

The Programs worksheet has the following toolbar buttons:

Send — Send any changes to the controller. The button is highlighted in red when there are unsaved changes.

Cancel — Cancel any changes in the worksheet.

Edit — Open the Control-BASIC editor for the selected program.

Export — Export the worksheet data as a .csv file. It will be downloaded to downloads folder on your system.

This option is not available when the enforce change approval option is enabled on the Enterprise Website Security page.

Export As Excel — Export the worksheet as an Excel file. It is downloaded to the downloads folder on your system.

An exported Excel file is digitally signed using the signature certificate specified on the Enterprise Website Security page.

Toggle Columns — Select which columns are displayed in the Programs worksheet.

Add — Open the Create Program dialog box to add a new program.

Delete — Delete a program.

 

The Programs worksheet contains the following columns:

Instance

The numbered instance of the program in the worksheet. Click the number to open the Control-BASIC editor for that program.

Name

A 32-character name.

Run

Use to enable or disable execution of the program.  Selecting the check box causes the program to be executed once per scan, clearing the check box prevents the execution of the program.

Auto/Manual

Use to toggle between Auto and Manual states. When a Control-BASIC program is placed in the Manual state, the Run value cannot be changed by Control-BASIC programs or the value of the associated Timer column.

Size

Indicates the size of the compiled program in bytes.

Exit

Indicates if the program was completely executed in a single scan. A No value in the Exit column means the program executed completely and a Yes value means the execution of the program was interrupted because it was too long for a single scan. Programs that have a Yes in the Exit column should be investigated for programming errors such as infinite loops. Each program is allowed a maximum of 200 ms to execute a scan. If a program contains code that requires longer than 200 ms to execute, the controller firmware interrupts the execution of the program and exits in order to allow other programs to be executed. The paused program resumes scanning from the place it was interrupted during the previous scan.

Description

BACnet description property, allowing up to 255 characters.
