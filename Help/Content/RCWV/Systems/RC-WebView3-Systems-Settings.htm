System Settings page

Modify the connection to a system using the Settings page.

To access the Settings page, click the name of the system in the System Tree and click Settings on the flyout menu.

The following buttons display in the toolbar:

Save — Saves any changes you make to the settings.
Cancel — Reverts any unsaved changes.

The Settings page contains the following options:

System Properties

IP Address

The IP address that connects RC-Studio to a Reliable Controls or third-party BACnet device, or RC-RemoteAccess server. This address is used as the primary point of access to a system. Typically, this is a web-enabled Reliable Controls system controller though it could also be a router or third-party BACnet device.

Protocol Type

Select the protocol used by the target system: BACnet, RCP, Dual, or B/VPN.

  • BACnet (default) — Only unencrypted BACnet protocol is used.
  • RCP — Only Reliable Controls Protocol is used.
  • Dual — Both BACnet or Reliable Controls Protocol (RCP) can be used without reconnecting. You can switch the System Tree view between BACnet and RCP views. Dual gives priority to RCP communication and cannot be used to access systems that do not use RCP.

  • B/VPN (BACnet Virtual Private Network) — RC-Studio connects to an RC-RemoteAccess server using encrypted BACnet protocol. The RC-RemoteAccess server must have a connection to one or more BACnet systems.

    When you select B/VPN, the Manage Certificates button appears in the System Properties area.

    1. Click Manage Certificates to open the Manage Certificates dialog box.

    2. Click the ellipsis "..." to browse, locate, and select a certificate,

      — or —

      Click Default to use the default RC-WebView certificate.

    3. Click OK to apply the certificate.

B/IP Port

Enter the port number the target system uses for BACnet over IP communications on the LAN or WAN which RC-WebView is connecting through.

System Access Username

Type a valid user name for the connected system. This is required before backing up or restoring panel files.

System Access Password

Type a valid password associated with the above user name. This is required before backing up or restoring panel files.

Job File

RC-WebView requires all graphics and site related files that are generated by RC-Studio in order to build the web pages for this system. The RC-Studio workstation used to generate the database for the original system, stores these files in the job directory in the RC-Studio Systems List. Click the ellipsis ... to open a file directory.

If these files reside on another computer then copy the entire Job Directory to the RC-WebView server. RC-WebView finds the files it requires when the job file is identified as the "label name" of the system in the Systems List, with an INI extension. For example:

C:\RC-Studio\<JobDirectory>\<Job>.ini

You also need to copy the Program Data folder for RC-Studio to the RC‑WebView server so that custom units and decimal places defined in RC-Studio are available for RC‑WebView. The folder is:

C:\ProgramData\Reliable Controls\RC-Studio

Important: Every time significant changes are made to the System using RC-Studio (e.g. Add/Delete objects or graphics), the entire job directory must be resaved to the RC-WebView server and the <job>.ini file must be rediscovered and the settings page resaved.

RC-Archive Properties

Enable RC-Archive Connection

Select the check box to allow RC-Archive to gather historical data for the current system.

SQL Server Name

The IP address or host name of the SQL server. RC-WebView, RC-Archive, and Microsoft SQL Server may run on different servers.

Database Name

The database name used by RC-Archive on the SQL server to store archived data.

User Name

The user name required to access the SQL database. By default, this is the "sa" account, otherwise it may have been changed during RC-Archive configuration.

Password

The password associated with the SQL user name.

Advanced BACnet

Use the settings below to edit the information in the Advanced BACnet Settings area.

Controller Device ID

The BACnet Device ID for the controller. Default is (auto). This must be set to the ID of a controller that supports B/SN connections.

RC-WebView Device ID

This is the BACnet device ID used by the RC-WebView server on the System. When the default setting is left in place the server chooses a random seven digit number. Or type a BACnet device instance that is unique for the System. Defaults to (auto).

Static BACnet Port

Used when third-party devices only support a fixed BACnet port.

Enable Fixed Port

Select to enable and edit the Static BACnet Port.

Enable RC-WebView File Server

This option is enabled by default and allows RC-Studio users to upload changes to RC-WebView through BACnet File Services. Disabling the option allows you to prevent unauthorized updates to the job files hosted on the RC-WebView File Server.

RC-WebView File Server Device ID

The device ID for the RC-WebView File Server instance. This only needs to be manually set to resolve an ID conflict.

RC-WebView File Server Network #

Sets the network number for the RC‑WebView File Server device. The network number needs to be unique within the system and not an existing BACnet network number. The default network number for the RC‑WebView file server is 65518. If you need to specify a different network number, avoid setting it in the 6551x number range.

APDU Timeout(ms)

The time allowed for a complete BACnet packet to be received at the target BACnet device, and for the subsequent acknowledgment to be received at the originating BACnet device. Reliable Controls recommends the default value of 6000 ms (6 seconds). The default is 6000.

APDU Segment Timeout(ms)

The time allowed for a single segment of a segmented BACnet packet to be received at the target BACnet device, and for the subsequent acknowledgment to be received at the originating BACnet device. Reliable Controls recommends the default value of 5000 ms (5 seconds). The default is 5000.

APDU Retries

The number of times the packet is sent if an acknowledgment is not received. The default is 1.
