Add a system

To add a controller system to the Enterprise Website, complete Step 2 of the Add A Website wizard.

Note: The system must be previously created in RC-Studio. Once the Enterprise Website is configured, you will be able to add multiple systems to the Enterprise Website.

Using information found in the Systems List of RC-Studio, add a system to the Enterprise Website in Step 2 of the Add A Website wizard:

  1. In the Add System page, enter the IP Address that connects RC-WebView to a Reliable Controls or third-party BACnet device. For a RC‑RemoteAccess B/VPN connection, enter the IP address for the public side of the firewall or router that protects the local network for the RC-RemoteAccess server. This server must be configured for port forwarding for the B/VPN port (default 8443).

  2. Select the Protocol Type to match the protocol used by the target system: BACnet, RCP, Dual, or B/VPN. The system connection information updates to the information required for that connection.

    When you select B/VPN, the Manage Certificates button appears in the System Properties area.

    1. Click Manage Certificates to open the Manage Certificates dialog box.

    2. Click the ellipsis "..." to browse, locate, and select a certificate,

      — or —

      Click Default to use the default RC-WebView certificate.

    3. Click OK to apply the certificate.
  3.  Enter the system connection information required for the selected protocol. A listing of the possible boxes is as follows:
    • RC-WebView Device ID — The BACnet device ID used by the RC-WebView server on the system. Enter a BACnet device instance that is unique for the system. The default is (auto) which generates a random seven digit number.
    • Controller Device ID — This is the BACnet device ID that RC-WebView uses to source system parameters, including passwords. The default is (auto). This must be set to the ID of a controller that supports B/SN connections.
    • B/IP Port — Enter the port number the target system uses for BACnet over IP communications on the LAN or WAN that RC-WebView is connecting through.
    • Force BBMD — Forces to register with the BACnet Broadcast Management Device (BBMD) at the listed IP address when connecting to BACnet IP protocol systems. This is typically used when is connecting to a system through a standard VPN connection.
    • Connection Port — Enter the B/VPN port number (default 8443).
    • Static BACnet Port — Enter a static BACnet port if required for third party devices that only support fixed port BACnet communications.
    • Enable Fixed Port — Select to enable and edit the Static BACnet Port.
    • System Access Username — Type a valid user name for the connected system. This is required before backing up or restoring panel files.
    • System Access Password — Type a valid password associated with the above user name. This is required before backing up or restoring panel files.
    • B/VPN Name — Enter the B/VPN name. This is case sensitive and must be an exact match.
    • B/VPN Password — Enter the B/VPN password. This is case sensitive and must be an exact match.

    • Job File — The location where the system's job files are stored.

      These job files are the graphics and site related files that are generated by RC-Studio in order to build the web pages for this system. The RC-Studio workstation used to generate the database for the original system, stores these files in the job directory in the RC-Studio Systems List.

      If these files reside on another computer then copy the entire Job Directory to the RC-WebView server. RC-WebView finds the files it requires when the job file is identified as the "label name" of the system in the Systems List, with an INI extension. For example:

      C:\RC-Studio\<JobDirectory>\<Job>.ini

      You also need to copy the Program Data folder for RC-Studio to the RC‑WebView server so that custom units and decimal places defined in RC-Studio are available for RC‑WebView. The folder is:

      C:\ProgramData\Reliable Controls\RC-Studio

  4. Click Next to see the Step 3 of the wizard: the RC-Archive Properties dialog box.
