Edit an animation in a Navigation Group

You can edit an animation by changing the settings on the tabs in the Edit Animation pane. In some cases, you may need to edit the files used by the animation. See Edit files used by an animation for more information.

Complete the following steps to edit an animation on the Group page of a Navigation Group using the Edit Animation pane:

  1. Open a Navigation Group by clicking:

    • The node for the Navigation Group in the System Tree to open the Group page.
    • A Navigation Group included in another Navigation Group to open the Group page.
    • A Navigation Group in the Navigation Tree in the client view, and then click Edit to open the Group page.

    Click the Group button if the Group page does not open by default.

  2. Identify items that you can edit if they are not readily apparent:

    • Press and hold the p or n keyboard key to highlight all items available for editing.

  3. Move your pointer to the animation and click Edit to open the Edit Animation pane on the right.

    The Edit Animation pane includes up to four tabs labeled Properties, Objects, Attributes, and Help.

  4. Use the Properties tab to:

    • Change the animation by uploading or selecting another RCA file. Where possible, the existing settings are applied to the new animation.

    • Adjust the width and height of the animation by changing the corresponding values in the boxes.

      If you lock the aspect ratio, changing the width or height value automatically adjusts the corresponding height or width value to retain the aspect ratio of the animation.

      You can also resize the animation by dragging a resize handle on a side or a corner of the animation. The animation retains its aspect ratio if it is locked on the Properties tab.

    • Lock or unlock the aspect ratio to indicate whether or not to preserve the aspect ratio when you resize the animation.

    • Adjust the X and Y coordinates to reposition the animation.

      You can also move the animation by holding the left mouse button when the pointer changes to a cross, and dragging it to another location.

    • Click Delete Animation to remove it from the Navigation Group.

  5. Use the Objects tab to specify one or more of the following. The number of columns on the tab is dependent on the animation.

    • Latitude — Click in the cell and specify the latitude for the map marker.

    • Longitude — Click in the cell and specify the longitude for the map marker.

    • Linked Object — Use the plus icon to open the Object Selection panes, and drag an object to the Linked Object column.
    • Display Name — Click in the cell and type the name to display when the pointer hovers over the animation.
    • Click Destination — Use Plus in the column to open a menu containing Object and Keyword. Click one of the items to open its respective selector panes, and then drag the object or keyword to the Click Destination column. A click destination requires that the object is specified in the Linked Object column.

    When an object is included in the table as a linked object or a click destination, click the minus icon to remove it.

  6. Use the Attributes tab to modify the attributes associated with animation, such as colors, fonts, and headings.

    The attributes included on the tab are dependent on the animation and some simple animations have no attributes.

  7. Use the Help tab to view details about configuring and using the animation.

Edit files used by an animation

You can upload files used by animations from your local workstation by specifying the files on the Attributes tab in the Edit Animation pane. For example, you can upload the .js file containing the rules and tasks defined for an IFDD animation, or you can upload the .json file used to define attributes for a FlexTile animation. If necessary, you can manually edit the uploaded .js or .json file on the RC‑WebView server in the Animations folder. The default location is:

C:\inetpub\wwwroot\Reliable Controls\RC-WebView\Uploads\Animations

Alternatively, you can edit the .js or .json file on your local workstation and reload the file through the Attributes tab to replace the file previously uploaded to the RC‑WebView server.
