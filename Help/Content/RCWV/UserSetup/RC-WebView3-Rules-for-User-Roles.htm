Rules for user roles

There are several important rules to keep in mind when working with user roles.

Applying a role to a user

  • If you apply a role to a user, then the role completely overwrites that user's permissions and settings and replaces them with the role's permissions and settings.
  • If a user selects or clears a permission setting that is defined in the assigned role, a Confirm Update Action message appears indicating that saving the change in permission settings opts the user out of all roles. If the user confirms the change, their permissions are set to those that are displayed on the Permissions page. The permissions may be different than those originally set for the user.
  • If a user has a Navigation Tree applied, and you apply a user role that doesn't have a default Navigation Tree, then the user keeps their Navigation Tree.
  • If a user has an After Hours Template applied, and you apply a user role that doesn't have a default After Hours Template, then the user keeps their After Hours Template.
  • If a user has a Home Group, and you apply a user role that doesn't have a default Home Group, then the user keeps their Home Group.
  • If a user has a myControl Home Group applied, and you apply a user role that doesn't have a default myControl Home Group, then the user keeps their myControl Home Group.
  • If a user has All Permitted Systems but inherits a role that restricts the user to just one system, then the user is restricted to just one system.
  • If you select Force All Systems for a user role in the Role Permissions page, then the users inheriting that role will always have access to all systems, no matter what other roles are applied.
  • If you select Force All Schedules for a user role in the Role Permissions page, then the users inheriting that role will always have access to all Schedules, no matter what other roles are applied.

Applying multiple roles to a user

  • If you apply multiple roles to the same user, then the user gets all of the permissions and settings from each of the roles. They add to each other, rather than cancel each other out.
  • If you apply multiple roles to the same user and there are conflicting Navigation Trees, After Hours Report Templates, or Home Groups, then the role that was most recently added to the user is applied.
  • If you apply multiple roles to the same user and there are conflicting access levels applied in Permissions, then the highest inherited access level is applied.
  • If you apply multiple roles to the same user and there are conflicting amounts of Auto Log Off minutes applied in Permissions, then the lowest amount of minutes is applied.

Removing and disabling roles

  • If you remove a user's roles, then their settings revert to their local user settings.
  • If you disable a user role, then it will not apply any settings to users until it is enabled.
  • If you apply a role to a user and then change a permission for that user, the user is removed from all applied roles, and reverts to whatever permissions they had previously.
