Audit Trail worksheet

The Audit Trail worksheet lists all the changes made by all users on the RC-WebView server. Depending on where you are in the interface, click either Options > Enterprise Website name > Audit Trail or Options > Audit Trail to view the Audit Trail worksheet.

When Enforce Change Approval is selected for the enterprise website:

  • All entries in the audit log are digitally signed to ensure their authenticity.
  • When viewing the Audit Trail, the Validated column includes checkmarks for entries with a valid signature.
  • When exporting the Audit Trail as an Excel file, the file contains a Data sheet and a Signature Sheet.

See Title 21 CFR Part 11 compliance for additional information.

The Audit Trail worksheet includes entries for:

  • Adding or modifying:
    • Enterprise Sites and Systems.
    • Users or Roles.
    • After Hours Templates.
  • Editing Schedules.
  • Creating Enterprise Schedules or changing the Schedule Tree within an Enterprise Schedule. It does not include schedule Events.
  • Acknowledging alarms.
  • Adding, modifying or removing Trend Logs or Control-BASIC programs.
  • Changes to Points.
  • Logon records.

Effective with RC-WebView 3.12.2, event data in the Audit Trail worksheet is automatically filtered in relation to what the user is viewing, that is, filtering by context. For example, if the user is viewing a System Group, the Audit Trail worksheet shows changes to all of the points included in the System Group.

Audit Trail entries prior to the installation of RC-WebView 3.12.2 are not included when entries are filtered. To view all previous and current entries, click Clear All Filters to remove the filter.

Filtering by context is applied when you open the Audit Trail worksheet while viewing the following pages:

  • Enterprise Schedules
  • System Groups
  • Workstation Groups
  • Navigation Groups
  • RCP Weekly and Annual Schedules
  • Runtime Reports and Logs
  • Trend Logs
  • Multipoint and Single-point Trend Log worksheets
  • Point Report
  • BACnet Schedules and Calendars

When you open the Audit Trail worksheet while viewing a page that uses the RCP protocol, such as an RCP Weekly Schedule or an RCP Runtime Report, the filtered entries identify the corresponding Panel but not the specific objects.

The Audit Trail worksheet contains the following columns:

Time

The specific time that events occurred.

User

Events that occurred when a specific user was logged on.

IP Address

The IP address of the user where the events occurred.

Point Name

Changes made to a specific point. Type the descriptor name or object name for a point.

Action

Events that included a specific action such as Logged in, Acknowledged Alarm, or Changed Analog Output.

Details

Lists events that occurred with specific details.

Reason

Displays the entry added by the operator in the Reason text box of the Edit Point dialog box.

System

Indicates the system ID associated with the entry in the Audit Trail.

Validated

Checkmarks indicate that the entries are validated with a signature.

Actions available for the Audit Trail

When you view the Audit Trail worksheet, you can:

  • Sort entries in a column in ascending or descending order by clicking the column heading. To ensure that the most recent entries are displayed at the top of the worksheet, click the Time column heading.
  • Adjust the width of a column by dragging its border left or right.
  • Reposition a column by dragging the column heading to a new position on the worksheet.
  • Change which columns are visible in the worksheet by clicking the Toggle Columns icon to open a popup window listing the available column names. Clear the check boxes to hide any columns that are not required. All columns are visible by default.
  • Export the Audit Trail worksheet as a CSV file by clicking the Export icon in the toolbar.
  • Remove all filters or context-based filters applied to the worksheet and display all audit log entries by clicking the Clear All Filters icon in the toolbar.
  • Filter records in the worksheet by clicking the Filter icon in the toolbar to open the Filter Records dialog box. Enter values in the input boxes to filter the records displayed. The labels for the input boxes match the column names in the Audit Trail worksheet.
    • To filter event data on a exact match to an entry in an input box, select the Exact check box.
    • To exclude all data events that match an entry in an input box, select the Exclude check box.
    • To further refine the filtering of the log entries, enter from and to dates in the Date range input boxes, and from and to times in the Time Range input boxes.
