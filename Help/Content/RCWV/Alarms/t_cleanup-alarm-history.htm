Delete alarm history entries

If your installation uses SQL Server Express, your database has a 10GB size limit. Deleting alarm history entries helps to maintain a manageable database size and ensure that RC‑WebView remains accessible.

Use the Cleanup Configuration dialog box to delete alarm history entries by specifying the cleanup options to apply to the entries.

Click Clean Up Alarm History on the Alarm History worksheet toolbar to open the dialog box.

If you have access to all systems on the Enterprise Website, automatic and manual cleanup options are available in the dialog box, otherwise only the manual cleanup option is available.

Automatically deleting entries

The automatic cleanup settings are available to users who have access to all systems on a site and their Clean Up Alarm History permission is enabled.

Figure: Cleanup Configuration - Automatic and Manual

To set the options to automatically cleanup alarm history entries:

  1. Click Clean Up Alarm History on the Alarm History worksheet toolbar to open the dialog box.
  2. In the Automatic area:
    1. Click the Enabled radio button.

    2. Enter the number of months in the Delete records older than input box.

      This indicates that any entries older than the number of months specified will be removed from the database.

    3. Enter the time of day to start the cleanup operation in the Time of day input box. The default time is 03:00.

  3. Click OK to save your settings.

    The cleanup operation will run in the background at the specified time.

Manually deleting entries

The manual options are available to users for their permitted systems only and when their Clean Up Alarm History permission is enabled.

Figure: Cleanup Configuration - Manual only

To immediately perform a cleanup of alarm history entries:

  1. Click for Selected Systems and select one or more of your permitted systems in the list.

  2. Enter the number of months in the Delete records older than input box. The default number is 6 months.

    This indicates that any entries older than the number of months specified will be removed from the database.

  3. Click Clean Up Now.

    A message opens indicating that when you click Yes the operation will proceed. It also indicates that the cleanup process can take a long time and cannot be undone.

  4. When you click Yes, an Operation in Progress message indicates the progress of the number of records being deleted.

    You can dismiss the message and continue with other tasks while the cleanup operation runs in the background.

Alarm history cleanup considerations

You should be aware of the following items when you use the alarm history cleanup function:

  • The cleanup of alarm history entries only occurs on the Enterprise Website where it is configured and it does not affect the alarm history entries for another site.
  • If you set Delete records older than to 0 months, any records before the current day are removed. This applies to the automatic or manual cleanup operations.
  • The automatic cleanup operation runs only once per day. If the cleanup operation ran earlier on the current day, and you change the Time of day setting to a different time, the new time takes effect on the following day.
