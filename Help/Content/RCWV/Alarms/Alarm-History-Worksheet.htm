Alarm History worksheet

View all previous alarms and their information in the Alarm History worksheet. The worksheet columns can be adjusted for size, position, and sort order, similar to a spreadsheet, and set to display or hide parameters as required.

To navigate to the Alarm History worksheet, click Options in the menu, then click Alarm History.

The Alarm History worksheet includes the following toolbar items:

Filter by System Click and select one or more systems in the list to show only their alarms.
Filter — Opens the Filter Alarm History dialog box where you select the settings to filter the items displayed in the worksheet.
Clear All Filters — Removes any filters applied in the worksheet.
Load — Refreshes the list of alarms in the worksheet.

Export — Exports a .csv file containing the alarm history information listed in the worksheet.

This option is not available when the enforce change approval option is enabled on the Enterprise Website Security page.

Export As Excel — Exports an Excel file containing the alarm history information listed in the worksheet.

An exported Excel file is digitally signed using the signature certificate specified on the Enterprise Website Security page.
Toggle Columns — Opens the shortcut menu to choose the columns displayed in the Alarm History worksheet. Select or clear check boxes to either display or hide columns as required.
Clean Up Alarm History — Opens the Cleanup Configuration dialog box where you select the options to automatically or manually remove entries from the alarm history. The user's Clean Up Alarm History permission needs to be enabled for the availability of this option.

Worksheet columns

The Alarm History worksheet contains the following columns:

Alarm Time

The time and date the alarm was generated at the controller level.

Alarm Type

Corresponds to the alarm types defined on the Type tab in the Alarm Configuration dialog box in RC-Studio (Alarms > Alarm Configuration). Alarm types 0 and 1 are defined as system and general, but 62 other alarm types can also be configured.

Message

The message associated with the alarm.

Alarm

There are four different alarm states:

  • Alarm — Unacknowledged alarm text appears in white on a red background.
  • Ack — Acknowledged alarm text appears in red on a normal background.
  • Reset — An automatically resolved alarm text appears in green on a normal background. To clear the alarm, highlight the row and click Ack on the worksheet toolbar.
  • Clear — Alarm text appears in white on a normal background. After an alarm is cleared, the alarm no longer appears in the Alarms worksheet. View cleared alarms in the Alarm History worksheet.

Priority

Determines the relative importance of each alarm. The lower the priority number, the more critical the nature of the alarm. For example, alarms that involve life safety may be assigned a priority number of 1. There are 255 priority levels available.

Source

Displays the number of the controller or BACnet object that generated the alarm.

System

The source of the alarm's system.

Time acknowledged

The time the alarm was acknowledged.

Time Reset

The time the alarm condition no longer existed.

Action

The action defined in Control-BASIC. It can be disabled, prompted, or unconditional as defined in the Alarm Configuration dialog box.

Acknowledged by User

Identifies the user who acknowledged the alarm.
