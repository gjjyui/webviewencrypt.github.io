Frequently asked questions

There are several questions that IT professionals frequently ask about RC‑WebView.

What is RC‑WebView?

RC‑WebView is an enterprise-level solution for linking together multiple BACnet systems together, allowing centralized access for managing multiple building management systems.

What is the RC‑WebView default TCP port?

The RC-WebView default TCP port is port:8180. If the target server with Internet Information Services (IIS) is already using this port, select an alternate port when installing RC‑WebView.

How does RC‑WebView communicate with building automation systems?

RC‑WebView is designed to view and manage one or more Reliable Controls system, comprised of Reliable Controls controllers and software, communicating over a BACnet network. Reliable Controls protocol (RCP) is also supported.

Does RC‑WebView require access to the internet?

RC‑WebView requires an internet connection to download the installation software download. Once downloaded, RC‑WebView can be installed on a server without an internet connection.

Does RC‑WebView have regular updates and releases?

Reliable Controls regularly posts software updates for fixes and releases for new features for RC‑WebView. The updated software is available for download from the Reliable Controls Support Center.

Does RC‑WebView require any client plug-ins?

No client plug-ins are required when using the latest version of RC‑WebView with enhanced System Groups. Adobe Flash is required for legacy Flash animations. Previous versions of RC‑WebView (RC‑WebView2.0) required ActiveX.

Does RC‑WebView use Secure Socket Layer (SSL)?

SSL can be enabled manually within Internet Information Services (IIS).

Does RC‑WebView require Internet Information Services (IIS)?

Yes. IIS is installed by the RC‑WebView installer as part of the installation process, and IIS is required for RC‑WebView operation.

Does RC‑WebView use single sign-on?

In the context of RC‑WebView, single sign-on refers to the ability to connect to multiple BACnet systems simultaneously using the RC‑WebView application. It also includes using Microsoft Active Directory for automated user credential management.

How does RC‑WebView store password information?

User passwords are encrypted and stored within the Microsoft SQL Server database. When Lightweight Directory Access Protocol (LDAP) is enabled, the users are not hosted by RC‑WebView, and instead reside as objects within an Microsoft Active Directory database.

What are the user-management options for RC-WebView?

RC‑WebView can connect using LDAP to a Microsoft Active Directory database. This type of user management is ideal for larger organizations.

What web technologies are required for RC‑WebView operation?

The most current version of RC‑WebView uses JavaScript and HTML5. Legacy versions of RC‑WebView also used Adobe Flash.

Where is RC‑WebView configuration data stored?

RC‑WebView configuration data is securely stored within the Microsoft SQL Server database. This database can be backed up to preserve the settings.
