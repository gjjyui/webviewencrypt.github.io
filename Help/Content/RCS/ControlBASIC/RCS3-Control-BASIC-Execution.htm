Execution

The runtime interpreter in a controller executes the compiled program line by line in a logically ordered process. Control-BASIC execution order begins with the first line of the first program and continues, line by line, through the last line of the last program. The controller's physical outputs are not assigned until after the very last line in the very last program is executed. This prevents accidental short cycling of an output in case different values are assigned to an output by various programs. All other values are assigned as each line is executed.

Control-BASIC programs

With each scan, the controller processes all Control-BASIC programs configured to Run. Each Control-BASIC program is allocated a maximum of 200 ms (0.2 s) to completely execute. If the allotted time expires and the Control-BASIC program has not completed execution, the controller pauses the program and exits. On the next controller scan, the Control-BASIC program continues from where it exited on the previous scan. This discontinuous operation is indicated to the programmer by a Yes value in the Exit column of the Control-BASIC worksheet.

Program functions

Compiled Control-BASIC programs are executed by the controller. Each scan starts with the lowest numbered program, typically Program 1 (PRG1), and ends with the highest numbered program in each controller.

Control-BASIC programs manipulate database elements as follows:

Time

The first elements updated in a Control-BASIC program are timer functions, such as TIME-ON, TIME-OFF, and INTERVAL. Real-number values are updated regardless of whether the line that contains the function is executed.

Local variables

Unique to each program; processed on a line-by-line basis.

Global variables ( VARS )

Retain the value set by the highest numbered program.

Analog outputs

Processed on a line-by-line basis, but the value is not written to the output until the end of the scan. An analog output value can be indexed and modified several times. The last value is written to the output in the next step. For example: PRG1 contains a line that increments Analog Output 1 (AO1) by 1; PRG4 contains a line that increments AO1 by 1. At the end of the scan, the value of AO1 has been incremented by 2.

Binary outputs

Updated and written to at the end of the scan. It is not valid to modify a binary output value and then use the modified value in a subsequent Control-BASIC statement. In addition to the above rules, all output and value variable objects have priority arrays. Values are written to the priority array of the object rather than directly to the present value. The object then defines its own present value based on the written value at the highest priority.

16 priorities

  • The highest priority is 1, and 16 is the lowest. By default, values are automatically written to an object's priority array at 10.
  • Command values may be written at other priorities in Control-BASIC as specified by the SET-PRIORITY statement.
  • Priority 10 is reset at the start of each program with firmware version 7.20 or later.
  • The present value of these objects is evaluated on a line-by-line basis to enable subsequent lines and programs to use these values.

Note: Physical outputs are still not defined until the last line of the last Control-BASIC program is executed.
