Control-BASIC compiler errors

The following error messages may be generated by the Control-BASIC compiler when you send a program to a controller:

Syntax error messages

Error No. Message Explanation

1

N/A

A point (variable) is missing or entered incorrectly.

2

N/A

The equal (=) sign is missing from a statement.

3

N/A

An operator is missing or entered incorrectly in an expression.

4

N/A

An error in the way brackets are entered.

5

N/A

Attempt to assign a value to an expression.

6

N/A

A function argument was not an expression.

7

N/A

A typing mistake.

8

N/A

Statement or function requires another expression.

9

GOTO or GOSUB to undefined line xx

A GOTO statement references an unspecified line number.

10

N/A

The closing quote was not included in a string attached to a PRINT statement.

11

N/A

RCP point values may be obtained from a different controller, but any RCP point must be modified on the controller that hosts the point.

Structural error messages

Error No. Message Explanation

1

PROGRAM TOO LARGE

– TEXT BUFFER

The program exceeds the maximum size. Break the program into smaller pieces to compensate.

2

TOO MANY GOTOS

– ALLOWED 128

There are too many GOTO statements in the program. The maximum allowed is 128 per program.

3

PROGRAM TOO LARGE

AT LINE ###

The program exceeds the maximum length and must be reduced in size to be compiled. Consider breaking the program into two smaller pieces.

4

TOO MANY LINES

– ALLOWED 200

The program exceeds the maximum number of numbered lines. The maximum number of lines in a program is 200. Try combining single line statements to free up extra lines.

5

FOR WITHOUT NEXT

The FOR statement must be used in conjunction with the NEXT statement. A NEXT statement is missing.

6

GOTO OR GOSUB

TO UNDEFINED LINE ####

An error has been made in defining the destination branch of a GOTO or GOSUB statement.

7

NEXT WITHOUT

FOR IN LINE ####

The NEXT statement must be used in conjunction with a FOR statement. A FOR statement is missing.

8

TOO MANY FOR’S

– ALLOWED 10

Program uses too many FOR statements. Each program can contain a maximum of 10.

9

EXPECTING LINE NUMBER

AFTER LINE ####

A line number was not entered.

10

LINE AFTER #### IS TOO LONG

The specified line exceeds the maximum allowed length. Break the line into two or more lines.
